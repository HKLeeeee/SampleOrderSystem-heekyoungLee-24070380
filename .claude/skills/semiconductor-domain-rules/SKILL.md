# 반도체 시료 도메인 규칙 (고정 명세 — 변경 금지)

## 1. 주문 상태 머신
| 상태 | 의미 | 진입 조건 |
|---|---|---|
| RESERVED | 주문 접수 | 주문 생성 시 초기 상태 |
| REJECTED | 주문 거절 | RESERVED에서 거절. 종결 상태, **모니터링 집계 제외** |
| PRODUCING | 생산 중 | 승인 시 재고 < 주문량 → 생산 큐 등록과 동시 전환 |
| CONFIRMED | 출고 대기 | ① 승인 시 재고 ≥ 주문량 ② 생산 완료 시 |
| RELEASE | 출고 완료 | CONFIRMED에서 출고 실행. 종결 상태 |

허용 전이: `RESERVED→{CONFIRMED, PRODUCING, REJECTED}`, `PRODUCING→CONFIRMED`, `CONFIRMED→RELEASE`
이 외의 전이는 모두 예외 처리한다.

## 2. 생산 계산 공식
```
부족분       = 주문량 − 현재 재고            (부족분 ≤ 0 이면 생산하지 않음)
실생산량     = ceil(부족분 / (수율 × 0.9))    ← 0.9는 오차 보정계수(상수화할 것)
총 생산시간  = 평균 생산시간 × 실생산량
```
검증 예시(명세 UI 기준): 부족분 170ea, 수율 0.92 → 실생산량 206ea, 0.8min/ea → 165min(반올림 표기)

## 3. 생산 라인
- 단일 라인, 한 번에 한 주문만 생산
- 스케줄링: **FIFO** (승인 순서대로)
- 주문이 들어온 시료만 생산 (선제/여분 생산 금지)
- 생산 완료 시: 재고 += 실생산량 반영 → 해당 주문 `PRODUCING→CONFIRMED`

## 4. 재고 상태 판정 (모니터링)
| 상태 | 조건 |
|---|---|
| 고갈 | 재고 == 0 |
| 부족 | 주문(미출고) 대비 재고 부족 |
| 여유 | 주문 대비 재고 충분 |
판정 우선순위: 고갈 → 부족 → 여유

## 5. 엔티티 필수 속성
- Sample: 시료 ID(`S-NNN`), 이름, 평균 생산시간(min/ea), 수율(0~1), 현재 재고(ea)
- Order: 주문번호(`ORD-YYYYMMDD-NNNN`), 시료 ID, 고객명, 수량, 상태, 생성/처리 일시
- 시스템에 등록된 시료만 주문 가능

## 6. 출고 규칙
- CONFIRMED 주문만 출고 가능
- 출고 시: 재고 −= 주문 수량, 상태 `RELEASE`, 처리일시 기록
