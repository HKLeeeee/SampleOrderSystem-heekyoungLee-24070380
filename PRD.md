# PRD.md — 반도체 시료 생산주문관리 시스템 기능명세서

> 본 문서는 구현의 기준이 되는 요구사항 문서이며, 변경 시 코드보다 먼저 갱신한다.

---

## 0. 문서 개요

| 항목        | 내용                                                   |
| ----------- | ------------------------------------------------------ |
| 시스템명    | 반도체 시료 생산주문관리 시스템 (SampleOrderSystem)    |
| 운영 형태   | 콘솔 기반, 담당자가 직접 메뉴 번호/값 입력             |
| 사용자      | 주문 담당자(접수), 생산 담당자(승인·거절·출고), 관리자 |
| 핵심 도메인 | 시료(Sample), 주문(Order), 생산 라인(단일, FIFO), 재고 |

### 0.1 용어 정의

- **시료(Sample)**: 시스템의 기본 단위. 등록된 시료만 주문 가능
- **수율(yield)**: 정상 시료 / 총 생산 시료 (예: 100개 생산 중 정상 90개 = 0.9)
- **부족분**: 주문량 − 현재 재고
- **실생산량**: `ceil(부족분 / (수율 × 0.9))` — 0.9는 오차 보정계수
- **총 생산시간**: 평균 생산시간 × 실생산량

### 0.2 주문 상태 정의 (전 기능 공통)

| 상태      | 의미                              | 비고                                 |
| --------- | --------------------------------- | ------------------------------------ |
| RESERVED  | 주문 접수                         | 초기 상태                            |
| REJECTED  | 주문 거절                         | 정상 흐름 외 상태, **모니터링 제외** |
| PRODUCING | 승인 완료 + 재고 부족으로 생산 중 | 생산 큐 등록과 동시 전환             |
| CONFIRMED | 승인 완료 + 출고 대기             | 재고 충분 승인 또는 생산 완료 시     |
| RELEASE   | 출고 완료                         | 종결 상태                            |

허용 상태 전이:

```
RESERVED → CONFIRMED | PRODUCING | REJECTED
PRODUCING → CONFIRMED
CONFIRMED → RELEASE
```

이 외의 전이는 모두 오류로 처리한다.

## FR-0. 메인 메뉴

### 개요

기능별 선택 화면을 Display 하고, 전체 시료에 대한 요약 정보를 표시한다.

### 표시 정보 (요약 대시보드)

| 항목      | 산출 기준                |
| --------- | ------------------------ |
| 등록 시료 | 등록된 시료 종 수        |
| 총 재고   | 전체 시료 재고 합계 (ea) |
| 전체 주문 | 누적 주문 건수           |
| 생산라인  | 생산 큐 대기 건수        |
| 현재 일시 | 시스템 시각              |

### 메뉴 구성

```
[1] 시료 관리   [2] 시료 주문   [3] 주문 승인/거절
[4] 모니터링   [5] 생산라인 조회   [6] 출고 처리   [0] 종료
```

### 처리 규칙

- 메뉴 선택 → 해당 기능 수행 → 완료 후 메인 메뉴 복귀 (루프)
- `[0]` 입력 시 데이터 저장 보장 후 종료

### 예외

- 범위 밖 번호/숫자 아닌 입력 → 오류 안내 후 재입력

### 인수 조건 (AC)

- [ ] 메인 화면에 요약 4종(시료 수, 총 재고, 주문 수, 대기 건수)이 표시된다
- [ ] 6개 기능 + 종료로 진입/복귀가 가능하다
- [ ] 잘못된 입력에도 프로그램이 종료되지 않는다

## FR-1. 시료 관리

### FR-1.1 시료 등록

- **입력**: 시료 ID, 이름, 평균 생산시간(min/ea), 수율
- **처리 규칙**
  - 시료 ID 중복 불가
  - 수율은 0 초과 1 이하, 평균 생산시간은 0 초과
  - 등록 시 초기 재고는 0 (또는 입력값 — 구현 시 결정하되 문서화)
- **출력**: 등록 완료 메시지 + 등록된 시료 정보
- **예외**: 중복 ID, 범위 밖 수율/시간, 빈 이름 → 오류 안내 후 재입력

### FR-1.2 시료 조회

- **처리 규칙**: 등록된 모든 시료 목록 + **현재 재고 수량 함께 표시**
- **출력 컬럼**: ID, 시료명, 평균 생산시간, 수율, 현재 재고
- **부가**: 목록이 길 경우 페이지네이션( [N] 다음페이지 ) 지원
- **예외**: 등록 시료 0건 → "등록된 시료가 없습니다" 안내

### FR-1.3 시료 검색

- **입력**: 검색어 (이름 등 속성)
- **처리 규칙**: 이름 부분 일치 검색을 기본으로 하고, ID 검색 지원
- **출력**: FR-1.2와 동일 컬럼의 결과 목록
- **예외**: 결과 0건 → 안내 메시지

### 인수 조건 (AC)

- [ ] 등록한 시료가 조회 목록에 즉시 나타난다 (재실행 후에도 유지)
- [ ] 중복 ID 등록이 거부된다
- [ ] 수율 1.5, -0.1 같은 비정상 값이 거부된다
- [ ] 이름 일부("웨이퍼")로 복수 시료가 검색된다

## FR-2. 시료 주문 (예약 접수)

### 개요

고객 요청에 따라 주문 담당자가 주문을 생성한다. 접수 시점 상태는 `RESERVED`.

### 입력

| 항목      | 검증               |
| --------- | ------------------ |
| 시료 ID   | 등록된 시료여야 함 |
| 고객명    | 공백 불가          |
| 주문 수량 | 양의 정수          |

### 처리 규칙

1. 입력 내용 확인 화면 표시 (시료명 포함) → `[Y] 예약 접수 / [N] 취소`
2. 접수 시 주문번호 발급: `ORD-YYYYMMDD-NNNN` (일자별 순번, 중복 불가)
3. 상태 `RESERVED` 로 저장
4. **재고 확인/승인은 이 단계에서 수행하지 않음** (FR-3에서 처리) — 안내 문구 표시

### 출력

- 접수 완료 메시지, 주문번호, 현재 상태 배지(RESERVED)
- "재고 확인은 [3] 승인 메뉴에서 진행" 안내

### 예외

- 미등록 시료 ID → 오류 안내 (시료 검색 유도)
- 수량 0/음수/숫자 아님 → 재입력
- [N] 취소 → 저장 없이 메인 복귀

### 인수 조건 (AC)

- [ ] 미등록 시료 ID로는 주문이 생성되지 않는다
- [ ] 접수 직후 상태가 RESERVED이고 FR-3 목록에 나타난다
- [ ] 주문번호가 형식에 맞고 중복되지 않는다

## FR-3. 주문 승인/거절

### 개요

`RESERVED` 주문 목록을 확인하고, 특정 주문을 승인 또는 거절한다.

### FR-3.1 접수 주문 목록

- RESERVED 상태 주문만 표시
- 컬럼: 번호, 주문번호, 고객, 시료, 수량, 상태
- RESERVED 0건 → "승인 대기 주문이 없습니다" 안내 후 복귀

### FR-3.2 주문 승인 — 재고에 따른 자동 분기

승인할 번호 선택 시 재고 확인을 수행하고 다음과 같이 **자동 분기**한다.

**(a) 재고 충분 (재고 ≥ 주문량)**

- 즉시 `RESERVED → CONFIRMED`
- 재고 예약 처리(권장: 출고 시 차감하되, 중복 승인으로 재고 초과 약속이 없도록 가용 재고 관리 — 구현 방식 문서화)

**(b) 재고 부족 (재고 < 주문량)**

- 부족분 / 실생산량 / 총 생산시간을 계산하여 표시
  - `부족분 = 주문량 − 재고`
  - `실생산량 = ceil(부족분 / (수율 × 0.9))`
  - `총 생산시간 = 평균 생산시간 × 실생산량`
  - 검증 예: 재고 30, 주문 200, 수율 0.92, 0.8min/ea → 부족 170 → 실생산량 206ea / 165min
- `[Y] 승인 / [N] 주문 거절` 확인
- 승인 시: **생산 라인(큐)에 자동 등록** + `RESERVED → PRODUCING`
- 상태 변경 결과 표시 (예: RESERVED → PRODUCING, 주문번호)

### FR-3.3 주문 거절

- 선택 주문을 즉시 `RESERVED → REJECTED`
- 거절 사유 입력은 선택 사항(자율)

### 예외

- 목록에 없는 번호 선택 → 재입력
- 이미 처리된 주문 → 오류 안내

### 인수 조건 (AC)

- [ ] 재고 충분 주문 승인 시 즉시 CONFIRMED가 된다
- [ ] 재고 부족 주문 승인 시 PRODUCING + 생산 큐에 FIFO로 등록된다
- [ ] 부족분/실생산량/생산시간이 공식대로 표시된다 (170→206 검증)
- [ ] 거절 시 REJECTED가 되고 이후 모니터링 집계에서 제외된다

## FR-4. 모니터링

### 개요

주문·재고 현황을 한눈에 파악한다. **REJECTED는 유효 주문이 아니므로 무시.**

### FR-4.1 주문량 확인

- 상태별 건수 표시: RESERVED / CONFIRMED / PRODUCING / RELEASE
- PRODUCING에는 "생산라인 대기" 등 보조 표기 가능
- REJECTED는 표시·집계 모두 제외

### FR-4.2 재고량 확인

- 시료별: 시료명, 재고(ea), 상태, 잔여율(%)
- **상태 판정 규칙** (우선순위 순):
  | 상태 | 조건 |
  |---|---|
  | 고갈 | 재고 == 0 |
  | 부족 | 미출고 주문 수요 대비 재고 부족 |
  | 여유 | 주문 대비 재고 충분 |
- 잔여율 = 재고 / (재고 + 미충족 수요) 등 일관된 기준으로 산출하고 산식을 코드 주석/README에 명시 (UI 표기 수준 자율)

### 인수 조건 (AC)

- [ ] 상태별 건수 합계에 REJECTED가 포함되지 않는다
- [ ] 재고 0인 시료가 '고갈'로 표시된다
- [ ] 주문 수요가 재고를 초과하는 시료가 '부족'으로 표시된다

## FR-5. 생산라인 조회

### 개요

단일 생산 라인의 현재 생산 정보와 대기 큐(FIFO)를 표시한다.

### 전제 규칙 (생산 라인 동작)

- 생산 라인은 1개(단일 라인)이며, 한 번에 하나의 주문만 생산
- **주문이 들어온 시료에 대해서만** 생산 (선제 생산 금지)
- 스케줄링: **FIFO** — 승인(등록) 순서대로 처리
- 생산 완료 시: 재고에 실생산량 반영 + 해당 주문 `PRODUCING → CONFIRMED`

### FR-5.1 생산 현황 표기

- 현재 생산 중인 주문 정보 표시 (표기 수준 자율)
- 권장 항목: 주문번호, 시료, 주문량, 재고→부족분→실생산량(수율/시간), 진행률, 완료 예정 시각
- 생산 중인 작업이 없으면 IDLE 상태 표시

### FR-5.2 대기 주문 확인

- 생산 큐의 대기 목록을 FIFO 순으로 출력
- 권장 컬럼: 순서, 주문번호, 시료, 주문량, 부족분, 실생산량, 예상 완료 시각
- 산식 각주 표기 권장: `부족분 = 주문량 − 재고`, `실생산량 = ceil(부족분/(수율×0.9))`

### 생산 진행 모델 (구현 결정 사항 — 문서화 필수)

콘솔 단일 프로세스 특성상 다음 중 택1 하여 README에 명시:

- (A) 시간 시뮬레이션: 메뉴 진입/명령 시 경과 시간을 계산해 완료 여부 판정
- (B) 수동 진행: "생산 완료 처리" 명령으로 큐 선두부터 완료
- (C) 백그라운드 스레드: 실시간 진행률 갱신
  ※ 어떤 방식이든 FIFO 순서·완료 시 상태 전환/재고 반영 규칙은 동일하게 보장

### 인수 조건 (AC)

- [ ] 대기 목록이 승인 순서(FIFO)와 일치한다
- [ ] 생산 완료 시 주문이 CONFIRMED로 바뀌고 FR-6 목록에 나타난다
- [ ] 생산 완료 시 재고가 실생산량만큼 증가한다

## FR-6. 출고 처리

### 개요

재고가 충분해진 `CONFIRMED` 주문에 대해 출고를 실행한다.

### 처리 규칙

1. CONFIRMED 주문 목록 표시 (번호, 주문번호, 고객, 시료, 수량)
2. 출고할 번호 선택 → 출고 실행
3. 재고 차감: `재고 −= 주문 수량`
4. 상태 `CONFIRMED → RELEASE`, 처리일시 기록
5. 결과 표시: 주문번호, 출고수량, 처리일시, 상태 변경(CONFIRMED → RELEASE)

### 예외

- CONFIRMED 0건 → "출고 가능 주문이 없습니다"
- 출고 시점 재고 < 주문 수량 (이론상 발생 가능한 경합 상태) → 출고 차단 + 안내

### 인수 조건 (AC)

- [ ] CONFIRMED가 아닌 주문은 출고 목록에 나타나지 않는다
- [ ] 출고 후 재고가 정확히 주문 수량만큼 차감된다
- [ ] 출고 후 상태가 RELEASE이며 모니터링 집계에 반영된다
